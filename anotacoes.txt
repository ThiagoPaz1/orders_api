🛠️ DESAFIO BACKEND — SISTEMA DE PEDIDOS

📌 Introdução
Uma empresa fictícia em crescimento precisa de uma API eficiente para gerenciar pedidos. Sua missão é construir essa API de forma moderna, usando GraphQL, garantindo integridade, performance e boas práticas de engenharia.

---

✅ A API deve permitir:
• Cadastrar usuários
• Listar usuários e seus pedidos
• Cadastrar produtos
• Listar produtos
• Emitir ordens de compra de produtos

---

📋 Requisitos obrigatórios:
• Pedidos simultâneos devem ser processados corretamente, mantendo a integridade do estoque
• Se um pedido não puder ser concluído por falta de estoque, deve ser rejeitado com erro apropriado
• API disponibilizada via GraphQL
• Projeto com Dockerfile e docker-compose para execução
• API escrita preferencialmente em Node.js ou Golang
• Testes automatizados para as principais regras de negócio

⭐ Opcionais (diferenciais):
• Workflow configurado no GitHub Actions
• Logs estruturados

---

🔍 Critérios de avaliação:
• Decisões técnicas adotadas e suas justificativas
• Performance da API
• Arquitetura e organização do código
• Modelagem de dados e uso de transações
• Qualidade dos testes
• Clareza e completude da documentação
• Tratamento de erros e robustez

---

📦 Entrega esperada:
Um repositório (GitHub ou similar) contendo a API implementada e um README com:
• Instruções de execução
• Decisões técnicas
• Trade-offs considerados
• O que faria diferente com mais tempo

---

⏱️ Escopo e tempo estimado:
Este desafio foi calibrado para 4 a 6 horas. Não esperamos funcionalidades além das descritas. Se quiser sugerir melhorias, descreva no README em vez de implementá-las.

---

💡 Sugestão de estrutura de banco de dados (pode adaptar conforme preferir):

users: id, name, email, created_at
products: id, name, price, stock, created_at
orders: id, user_id, total, created_at
order_items: id, order_id, product_id, quantity, price
